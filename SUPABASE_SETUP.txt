Install the Supabase JavaScript client:

```bash
npm install @supabase/supabase-js
```

## Files Created

### Core Files
- `src/lib/supabase.ts` - Supabase client initialization
- `src/services/authService.ts` - Authentication operations
- `src/services/bookingService.ts` - Booking operations

## Usage Examples

### Authentication

```typescript
import { useAuth } from "@/hooks/useAuth";

function MyComponent() {
  const { user, isLoading, isAuthenticated } = useAuth();

  if (isLoading) return <div>Loading...</div>;

  return (
    <div>
      {isAuthenticated ? (
        <p>Welcome, {user?.prenom}!</p>
      ) : (
        <p>Please log in</p>
      )}
    </div>
  );
}
```

### Get User Bookings

```typescript

  );
```

### Search Voyages

```typescript
import { useVoyageSearch } from "@/hooks/useVoyages";

function SearchVoyages() {
  const [searchTerm, setSearchTerm] = useState("");
  const { voyages, isLoading } = useVoyageSearch(searchTerm);

  return (
    <div>
      <input
        type="text"
        value={searchTerm}
        onChange={(e) => setSearchTerm(e.target.value)}
        placeholder="Search voyages..."
      />

      {isLoading && <p>Searching...</p>}

      <div>
        {voyages.map((voyage) => (
          <div key={voyage.id}>{voyage.title}</div>
        ))}
      </div>
    </div>
  );
}
```

## Protected Routes

Use `useRequireAuth()` hook to protect routes:

```typescript
import { useRequireAuth } from "@/hooks/useAuth";

function ProtectedPage() {
  const { user, isLoading } = useRequireAuth();

  if (isLoading) return <div>Loading...</div>;

  return <div>Protected content for {user?.prenom}</div>;
}
```

## Error Handling

All service functions return an object with:

```typescript
{
  success: boolean;
  data?: any;
  error?: string;
}
```

Example error handling:

```typescript
const result = await createBooking(userId, bookingData);

if (!result.success) {
  toast.error(result.error || "Something went wrong");
  return;
}

toast.success("Booking created successfully!");
```

## Troubleshooting

**"Missing Supabase credentials" error:**
- Ensure `VITE_SUPABASE_URL` and `VITE_SUPABASE_KEY` are in your `.env` file
- Restart the dev server after adding env variables

**"relation does not exist" error:**
- Run your SQL schema from `database/schema.sql` in Supabase
- Check that the table names match your queries

**Authentication not persisting:**
- Supabase handles session persistence automatically
- Check browser console for errors
- Ensure cookies are enabled
